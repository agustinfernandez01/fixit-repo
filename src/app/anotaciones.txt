funcionamiento tokens

LOGIN
→ se crea sesión en Db
→ se dan access + refresh

REQUESTS
→ usan access (rápido, sin DB)

ACCESS EXPIRE
→ se usa refresh

REFRESH
→ backend valida:
   1. el token (JWT)
   2. la sesión (DB)

→ si todo ok → nuevo access

LOGOUT
→ se revoca sesión en DB
→ el refresh deja de servir
